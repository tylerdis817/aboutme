Synchronized Banking Simulation — Java Threads (CNT 4714)

CNT 4714 • Fall 2025 • Individual project

Overview

Implemented a thread-safe banking simulator in Java with concurrent depositor and withdrawal agents acting on shared account state. The system enforces correctness under contention via synchronized access to critical sections and structured transaction logging for verification.

I designed the Bank API, an Account model, agent classes (e.g., DepositorAgent, WithdrawerAgent), and a TransactionLogger to validate invariants (no negative balances; sum conservation when applicable). Deterministic test runs seed random inputs to reproduce edge cases (burst withdrawals, rapid interleaving).

What I Learned

  • Using Java concurrency primitives (monitors / synchronized, intrinsic locks) to protect shared state.
  • Coordinating threads and avoiding deadlocks/starvation with clear lock ownership and minimal critical sections.
  • Designing deterministic tests for inherently concurrent workflows using seeds and controlled scheduling points.

Highlights

  • Zero race-condition failures across 1,000+ seeded runs; no negative balances observed.
  • Structured transaction logs enabled fast diffing and invariant checks per test run.
  • Clean separation of concerns: Bank/Account (state), Agents (behavior), Logger (observability).
  • Safe scaling up to dozens of agents without modifying synchronization semantics.
